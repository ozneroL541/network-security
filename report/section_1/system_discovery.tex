\chapter{System Discovery}
\label{cha:system_discovery}
In this chapter we will describe how to perform a System Discovery using OpenVAS.
There must be clarified that this type of scan is not a Vulnerability Scan, 
it's just a detection of the targets of a system, their open ports, the services running on them and its whole topology.

\section{Get Debian VM IP address}
Before performing the System Discovery, we need to know the IP address of the Debian VM,
since we need to specify it as Greenbone default gateway to reach the docker network where the vulnboxes are located.
Open the Debian VM, which credentials are:
\begin{itemize}
    \item Username: root
    \item Password: root
\end{itemize}
Then go to the folder \texttt{~/network-security/project/docker-vulnet/} 
and run the command \texttt{bash hardcore-setup.sh} to burn down the firewall 
and build up some rules which allow the Debian machine to redirect the traffic 
to the docker network.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{../presentation/images/debian_start2.png}
    \caption{Debian VM - Initial setup}
    \label{fig:debian_setup}
\end{figure}

After that, run the command \texttt{ip addr | grep 192} to get the IP address of the Debian VM.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{../presentation/images/debian_ip.png}
    \caption{Debian VM - IP address}
    \label{fig:debian_ip}
\end{figure}

\section{Configure Greenbone Gateway}
Before performing the System Discovery, we need to configure the Greenbone Gateway to 
reach the docker network where the vulnboxes are located.
Open the OpenVAS VM, which credentials are:
\begin{itemize}
    \item Username: admin
    \item Password: admin
\end{itemize}

Go to \textbf{Setup>Network>Global Gateway} and change the default gateway 
according to the IP address of the Debian VM, which we got in the previous section.
Then save the configuration.

\section{System Scan}
Now go to the \textit{Scans} section and click on \textit{Tasks}.
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{../presentation/images/gs_sd_1.png}
    \caption{Greenbone Web Interface - Main page}
    \label{fig:web_interface}
\end{figure}

Modify the \textit{Docker Vulnet} task.
Edit it by choosing \textit{System Discovery} as the scan configuration, 
then save the task.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{../presentation/images/gs_sd_2.png}
    \caption{Greenbone Web Interface - Task modification}
    \label{fig:task_modification}
\end{figure}

Now we can launch the task pressing the \textit{Play} button.
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{../presentation/images/gs_sd_3.png}
    \caption{Greenbone Web Interface - Task launch}
    \label{fig:task_launch}
\end{figure}
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{../presentation/images/gs_sd_5.png}
    \caption{Greenbone Web Interface - Task progress}
    \label{fig:task_progress}
\end{figure}

After about 10 minutes, the task will be completed and we can see the results.
To see the results, click on the task and then on the \textit{Report} tab.
In the upper right corner, click on the \textit{Remove filters} button to see all the results.
This is required because by default the web interface shows only the results with a severity score, 
but since the \textit{System Discovery} scan configuration doesn't detect vulnerabilities, 
all the results have a severity score of 0, so we need to remove the filters to see them all.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.4\textwidth]{../presentation/images/gs_sd_4.png}
    \caption{Greenbone Web Interface - Remove filters}
    \label{fig:remove_filters}
\end{figure}

Now we can see all the results of the scan which comprehends:
\begin{itemize}
    \item The detected hosts in the given IP range
    \item The open ports on the hosts
    \item The services running on the open ports
    \item The topology of the system
\end{itemize}
Even if the scanned system only has 6 vulnboxes, as we can see, 
OpenVAS detected a lot more hosts, since in the OpenVAS view, each IP 
address corresponds to a host, and in the scanned system some vulnboxes 
have more than one IP address, therefore OpenVAS detected all of them as distinct hosts.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{../presentation/images/gs_sd_6.png}
    \caption{Greenbone Web Interface - Scan results}
    \label{fig:scan_results}
\end{figure}
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{../presentation/images/gs_sd_7.png}
    \caption{Greenbone Web Interface - Operating Systems}
    \label{fig:detected_hosts}
\end{figure}
